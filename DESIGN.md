# K문서툴 디자인 기준

K문서툴은 무료 문서 도구 모음이다. 첫 화면은 서비스 소개가 아니라 도구 실행 화면이어야 한다.

## 기준

- 목표: 사용자가 도구 이름이나 문제 상황을 보고 바로 실행한다.
- 성격: Cal.com처럼 깨끗하고 직접적인 제품 UI, IBM Carbon처럼 절제된 선과 타이포그래피.
- 밀도: 낮은 장식, 높은 스캔성. 행, 표, 검색, 버튼을 우선한다.
- 문장: 설명을 줄이고 명사형 라벨을 쓴다. 필요한 문장은 한 줄 안에서 끝낸다.

## 첫 화면

- H1은 `K문서툴`.
- K문서툴 위에는 작고 귀여운 문서 SVG 애니메이션을 둔다.
- 주요 문구는 실제 도구 수를 데이터에서 계산해 `설치 없이 쓰는 무료 도구 N종`으로 표시한다.
- 보조 문구는 설치 없음, 무료, 서버 전송 없음을 짧게 강조한다.
- 검색창은 첫 화면의 가장 강한 실행 요소다.
- 기본 검색 결과는 접고, 사용자가 입력하면 결과를 보여준다.
- 카테고리 필터는 `전체`, `업무 문서`, `PDF`, `이미지`.
- 문제 바로가기는 짧은 라벨로만 둔다. 예: `파일 형식 오류`, `용량 초과`, `여러 장 PDF`.
- 긴 소개, 사용법 설명, 기능 자랑, 가짜 통계는 넣지 않는다.

## 시각 시스템

- 배경: `#ffffff`
- 페이지 배경: `#f6f7f9`
- 표면: `#ffffff`
- 본문: `#111827`
- 보조 텍스트: `#667085`
- 선: `#d0d5dd`
- 약한 선: `#eaecf0`
- 기본 액션: `#111827`
- 포커스/선택 포인트: `#155eef`

## 형태

- 반경은 4px-8px를 기본으로 한다.
- 카드 그리드보다 행 목록을 우선한다.
- 도구 작업 화면은 입력과 결과만 먼저 보이게 한다.
- `처리 원칙`, `확인 사항` 같은 설명 패널은 첫 작업 영역에 노출하지 않는다.
- 큰 그림자, 그라데이션, 블롭, 장식 아이콘을 쓰지 않는다.
- 버튼은 누를 수 있어야 하고, 텍스트 링크처럼 숨기지 않는다.
- 아이콘은 얇은 단색 라인 아이콘으로 통일한다.

## 타이포그래피

- 기본 글꼴: `Inter, Pretendard, system-ui, sans-serif`.
- 폰트 크기와 굵기는 itool.co.kr의 스캔 가능한 도구형 UI에 맞춘다.
- 헤드라인도 과장하지 않는다. 도구 화면 안에서는 작업 밀도가 우선이다.
- 글자 간격은 0으로 둔다.

## 금지

- AI 템플릿처럼 보이는 대형 히어로, 중앙 정렬 카피, 둥근 카드 묶음.
- “올인원”, “스마트”, “강력한” 같은 범용 SaaS 문구.
- 기능을 설명하는 긴 문단.
- 의미 없는 대시보드, 가짜 KPI, 장식용 사이드바.
- 보라색/남색 그라데이션, 오브, 글로우.

## 체크

- 3초 안에 검색하거나 도구를 실행할 수 있어야 한다.
- 모바일에서도 첫 화면에 검색창과 실행 가능한 도구가 보여야 한다.
- 클릭 영역은 44px 이상이다.
- 모든 텍스트는 한눈에 스캔되어야 한다.
- 새 기능을 추가할 때는 `검색`, `행 목록`, `문제 바로가기` 중 어디에 들어갈지 먼저 정한다.
